3D Agentic System Architecture

Watch the interactive 3D animation on the left to see how our system components interact in real-time. You can click and drag the 3D scene to rotate the camera!

1. Orchestrator API (The Gateway)

What should we do? Handle incoming resumes instantly without blocking users during long AI parsing tasks.

How did we achieve it? The top Blue Cube represents the FastAPI Orchestrator. It receives user data, uses Gemini 2.0 to parse text, and pushes tasks down to the Database queue instantly. The flowing blue particles represent data tasks being queued.

2. Postgres Vector DB (The Core)

What should we do? Store tasks and candidate embeddings securely for semantic matching.

How did we achieve it? The central Green Cylinder is our Azure PostgreSQL DB. It holds the queue. The agents constantly orbit and poll this core for new tasks to claim.

3. Specialized Agent Fleet

What should we do? Decouple complex logic into isolated workers that scale independently (ATS, VMS, Matching, Submission, Sourcing).

How did we achieve it? The Purple Spheres orbiting the core are our agents. They claim tasks from the database asynchronously via the connecting tether lines, process them in isolation, and write the results back.

4. Self-Healing Agent

What should we do? Ensure failed tasks aren't lost and humans are alerted to deadlocks.

How did we achieve it? The Red Sphere rotating on a higher orbit represents the Self-Healing agent. It monitors the entire database queue from a bird's-eye view, looking for failed tasks, requeuing them, and sending alerts via Microsoft Graph if necessary.
